# ish v0.2 Design: "You could actually use this"

## Design Principles

- **POSIX compliant.** We meet the standard. Every syntactic position POSIX doesn't claim is ours. Atoms, tuples, `|>`, `fn`, `\->`, `do...end` — none violate POSIX because they occupy space POSIX never touched.
- **Everything is a value.** Processes, supervisors, functions, maps, tuples, lists, atoms — all first-class, all passable, all pattern-matchable.
- **TMTOWTDI.** `fn name do...end` or `name = fn do...end`. `\x -> x * 2` or `fn x do x * 2 end`. `if cond; then...fi` or `if cond do...end`. The language is syntactically contextual — it reads intent from position.
- **One pipe conceptually.** `|` and `|>` auto-coerce between bytes and values. Lines are the universal default. Explicit bridge functions (`from_json`, `from_csv`) override when the data isn't lines.
- **The kernel gives us Erlang.** Processes, isolation, signals — the primitives exist. ish makes them programmable with types and pattern matching.
- **Do one thing well.** ish owns typed data, pattern matching, closures, processes, supervision. Unix owns files, text transforms, process execution.

## What v0.1 delivered

- POSIX sh compatibility (commands, pipes, redirections, control flow, job basics)
- Typed data (atoms, tuples, lists, maps, pids, function values)
- Pattern matching and destructuring
- First-class functions (named, anonymous, multi-clause dispatch, guards, lambdas)
- Value pipes (`|>`) with transparent auto-coercion
- Lightweight processes with message passing (spawn, send, receive, selective receive, timeouts)
- Supervision trees (one_for_one, one_for_all, rest_for_one)
- Standard library (list/string/map transforms, bridge functions)
- Interactive REPL with history, tab completion, multi-line input

## v0.2 Scope

### 1. Job Control

**Problem:** ctrl-z backgrounds the shell itself, not the foreground child. `fg`/`bg` exist as builtins but terminal control isn't wired correctly.

**Fix:**
- Put child processes in their own process group (`setpgid`)
- Give the child process group the terminal (`tcsetpgrp`) before waiting
- Handle SIGTSTP: when the child is stopped, reclaim the terminal and report the job
- Handle SIGCONT for fg/bg resume
- Track job state (Running/Stopped/Done) correctly

**Behavior:** `./ish` → `nvim` → ctrl-z stops nvim, returns to ish prompt. `fg %1` resumes nvim. `bg %1` resumes in background.

### 2. `$()` in Double-Quoted Strings

**Problem:** `"$(cmd)"` does not currently substitute correctly. Users must use `#{expr}` or unquoted `$(cmd)`.

**Fix:**
- The lexer needs to recognize `$(` inside double-quoted strings and handle balanced parentheses
- The expansion code (`env.Expand`) needs to process `$(...)` substrings by running them through lex → parse → eval and substituting the result
- Nested `$()` must work: `"$(echo $(date))"`

**Behavior:** `echo "today is $(date +%Y-%m-%d)"` prints the date inline.

### 3. Script Safety

**Problem:** `set -e`, `set -u`, `pipefail`, and `trap ERR` are partially implemented. Edge cases exist.

**Fix:**
- `set -e`: exit on any command failure (non-zero exit code). Must NOT trigger inside `if` conditions, `&&`/`||` chains, or `!` negation. Must propagate through function calls.
- `set -u`: error on unset variable expansion. Already partially works but needs coverage in all expansion contexts.
- `set -o pipefail`: pipeline exit code is the last non-zero exit from any stage, not just the final stage.
- `trap ERR`: fire the ERR trap when a command fails (same rules as `set -e` for what counts as "fails").

**Behavior:** `set -e; false; echo "never reached"` exits after `false`.

### 4. String Interpolation Everywhere

**Problem:** `#{expr}` and `$var` work in double-quoted strings but not inside `$()` or in all heredoc contexts.

**Fix:**
- `$()` inside double-quoted strings (covered by item 2)
- `#{expr}` inside heredocs (unquoted delimiter)
- Ensure `$var` expansion is consistent in all string contexts where it should work

**Behavior:** All these work:
```
echo "hello $name"           # already works
echo "result: #{2 + 2}"     # already works
echo "date: $(date)"        # new
cat <<EOF
hello $name
result: #{2 + 2}
EOF                          # verify heredoc coverage
```

### 5. Float Math

**Problem:** ish only has integer arithmetic. No float type, no decimal literals, no float division.

**Fix:**
- **Lexer:** `5.0`, `3.14`, `.5` are `TFloat` tokens (digit(s) + dot + digit(s)). This interacts with the dot-path fix: `5.0` must be a float, not `5` + `.` + `0`. A dot between digits is part of a number.
- **AST/Value system:** Add `VFloat` kind to `core.Value`. Float literals produce `VFloat`.
- **Arithmetic:** Integer op integer = integer. Float op anything = float. Anything op float = float. `5 / 2 = 2` (integer division). `5.0 / 2 = 2.5` (float division).
- **Comparison:** Floats compare with integers naturally (`5 == 5.0` is `:true`).
- **Display:** `.String()` renders as `"3.14"`. Whole floats render as `"5.0"` (not `"5"`) to distinguish from integers.
- **JSON bridge:** `from_json` produces `VFloat` for non-whole numbers. `to_json` serializes floats correctly.

**Behavior:**
```
r = 5 / 2        # 2 (integer division)
r = 5.0 / 2      # 2.5 (float division)
r = 1 + 0.5      # 1.5
r = 3.14 * 2     # 6.28
```

## Implementation Order

1. Float math (self-contained, no dependencies on other items)
2. `$()` in strings + string interpolation (lexer/expansion work)
3. Script safety (eval-level, builds on existing flag infrastructure)
4. Job control (OS-level, most complex, most risk)

## Verification

Each item gets its own test suite. Full `go test -race ./...` after each. Manual REPL testing for interactive features (job control, ctrl-z). Example scripts updated to use new features where natural.
